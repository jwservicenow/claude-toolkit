## Structural findings — still open (as expected)

The republish addressed empty files only. The two structural items were raised but not committed by
the mirror team, and both persist — verified, not assumed:

- **Oversized monolithic indexes — *still open.*** Every bundle still has a single large `index.md`
  with no sub-indexes: Platform **1,214 KB**, ITOM **1,149 KB**, ITSM **1,011 KB**, ITAM **637 KB**.
- **Bundle-routing — *still open.*** The canonical MID Server documentation (**85 files**) remains
  under `servicenow-platform/mid-server/`, not the ITOM bundle. The MID-Server pages that do sit in
  ITOM are usage-specific tasks (Discovery, ACC, HLA), not the authoritative reference — so an
  ITOM-scoped query still won't reach the core content.

- **Sweep:** every `.md` under each bundle, sized via the GitHub git tree API on `australia`.
- **Cross-check:** tree-reported sizes compared against live `raw.githubusercontent.com` fetches —
  **0 mismatches across 15 sampled files.** No tree truncation on any bundle.
- **Stubs:** **zero** files in the 1–199 byte range anywhere — content remains binary (full or empty),
  same clean pattern as baseline.
